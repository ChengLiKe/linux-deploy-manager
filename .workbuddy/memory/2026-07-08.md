## Windows 适配重构（任务 7-15）
用户要求将代码从 Linux 专用重构为 Windows 可运行状态。

### 关键改动

1. **SQLite 驱动替换**：`mattn/go-sqlite3`（CGO）→ `glebarez/sqlite`（纯 Go），解锁 `CGO_ENABLED=0` 交叉编译。

2. **sysutil 跨平台工具包**：`internal/sysutil/` — 含 `ShellCommand`、`ShellCommandContext`、`ShellCommandWithSudo`、`DetachProcess`、`IsWindows`/`IsUnixLike`。Unix 用 `bash -c`，Windows 用 `cmd /C`。

3. **local_executor 重写**：`bash -c` → `sysutil.ShellCommandContext`；超时终止改为平台化 `terminateCmd`（Unix: SIGTERM→SIGKILL, Windows: taskkill）。

4. **deployer 适配**：
   - `killOldProcess`：Windows 本地用 `os.ReadFile` + `taskkill /T /F`
   - `killOldProcessLocalWindows`：新增函数
   - `mkdirRemote`：Windows 本地用 `os.MkdirAll`
   - `writeEnvFileRemote`：Windows 本地用 `os.WriteFile`
   - `checkRuntimeEnv`：Windows 工作目录用 `os.TempDir()`
   - `runLocalDeploy`：systemd → background 回退（Windows），env 包装跳过（Windows），background 模式走 `startBackground`（Windows 用 `CREATE_NEW_PROCESS_GROUP | DETACHED_PROCESS`），direct 模式跳过 bash pid 包装
   - `runCommandLegacy`：使用 `sysutil`，sudo 仅在 Unix 启用

5. **envman/service.go**：`os/exec` → `sysutil`，所有 `bash -c` 替换为 `sysutil.ShellCommand`

6. **instance_log_handler.go**：`journalctl` 仅 Unix（Windows 发 error）、`tail -f` 替换为 `powershell Get-Content -Wait`、`sudo bash -c` 替换为 `sysutil.ShellCommandContextWithSudo`（Windows 忽略）

7. **config/config.go**：默认路径跨平台化（Windows: `%APPDATA%/linux-deploy-manager`，Linux: `/var/lib/...`）

8. **git/service.go**：`/dev/null` → `os.DevNull`（跨平台）

9. **build.ps1**：Windows 构建脚本（CGO_ENABLED=0、Go 后端 + 前端构建 + Linux 交叉编译）

### 编译验证
- `CGO_ENABLED=0 go build` ✅
- `go vet ./...` ✅（无警告）
- 产出：15.8 MB Windows PE 二进制（`bin/linux-deploy-manager.exe`）

## SSH 密钥功能改进（任务 20-23）

4. **前端 KeyList 拆分帮助文案**：根据 `activeTab` 动态显示——Git tab 显示 Git 平台添加说明；服务器 tab 显示 authorized_keys 添加说明。
5. **前端导入密钥 UI**：创建表单新增「生成密钥」/「导入已有密钥」模式切换；导入模式含文件选择按钮、私钥/公钥文本输入框；取消时清空导入字段。
6. **API 层**：`api.ts` 新增 `keyApi.import()` 方法。

## Electron 自动更新
用户问「客户端自动更新怎么弄」。

### 实现方案
采用 **electron-updater + GitHub Releases** 方案，架构：
- `electron-updater` 检测 GitHub Releases 的最新发布，自动下载并提示安装
- `package.json` 配置 publish provider 为 github
- 新增 `.github/workflows/release.yml`：打 tag 自动触发全平台构建并上传到 Releases

### 改动
1. **安装 `electron-updater`**：npm install --save，v6.8.9
2. **`electron/main.js`**：集成 autoUpdater，含检查更新/下载进度/安装完成等事件；IPC handler 暴露 checkForUpdate/downloadUpdate/installUpdate
3. **`electron/preload.js`**：通过 contextBridge 暴露更新 API（checkForUpdate/downloadUpdate/installUpdate/onUpdateEvent）
4. **`package.json`**：新增 publish 配置（provider: github）；各平台 target 增加 publish
5. **`web/src/types/index.ts`**：ElectronAPI 接口增加更新方法；新增 UpdateEvent 接口
6. **`.github/workflows/release.yml`**：打 v* tag 自动触发，三平台（ubuntu/macos/windows）并行构建，产出发布到 GitHub Releases

### 使用方式
```bash
# 发布新版本
git tag v1.2.0
git push origin v1.2.0
# CI 自动构建并发布到 GitHub Releases
# 用户启动客户端后自动检查更新
```
